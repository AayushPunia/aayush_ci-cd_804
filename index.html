☁️ Cloud Computing
// What is it?
  • All tasks done by a virtual computer
  • Access through internet to compute task
  • All services accessible by computer in cloud
// Services (Type of Work)
  • IaaS — Infrastructure as a Service
    Hardware, processors, core resources
  • SaaS — Software as a Service
    Gmail — 0 to very less accessible
  • PaaS — Platform as a Service
    Dev tools, online compiler
  • FaaS — Function as a Service
    AWS Lambda, Firewall
🏗️ Deployment Models
// Security Level
  • Public — Access for all
  • Private — Authorized (username/pass)
  • Protected — Categorical access
  • Hybrid — Combination of 2 or more
  • Community — Shared by organisation · eg. Army
eg. IRCTC:
Train Schedule → Public
PNR Status → Protected
User Info → Private
Login as Agent → Community

Repository = folder · Directory = path · Stages = index
📂 Stages of Git
Workspace (Working Directory)
file viewing & editing · changes untracked · undo won't work
↓ git add
Staging Area (Index)
middle ground · no changes in file · sort necessary changes
↓ git commit
Local Repository (.git hidden folder)
once committed, work is permanently saved
↓ git push
Remote Repository (GitHub)
origin = local cache / bookmark of remote server
⌨️ Git Commands
// Upload
  • git add — move to staging
  • git commit — save to local repo
  • git push — send to remote
// Download
  • git fetch — get from remote
  • git merge — combine branches
  • git pull — fetch + merge
  • git rebase — rewrite history, linear commits
  • git pull --rebase — task similar to pull + linear
// Maintain
  • git init — initialize repo
  • git config — setup auth connection
  • git status — check current state
  • git remote add — connect local to remote
  • git log --oneline — see history
  • git restore — undo staging
